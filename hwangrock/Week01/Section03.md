## section03 - 스레드 생성과 실행

이 장에서는 java에서 스레드를 생성하고 실행하는 과정을 보여줍니다.  
Thread 클래스를 상속하는 방법과 runnable 인터페이스를 구현하는 방법으로 나눌 수 있습니다.  

---

### 스레드 생성 - Thread 클래스 상속

Thread 클래스를 상속받아 run() 메서드를 오버라이드하면 스레드에서 실행할 코드를 작성할 수 있습니다.  

스레드를 시작하려면 start() 메서드를 호출해야 하며,  
run()을 직접 호출하면 단순히 메서드 실행일 뿐 새로운 스레드가 생성되지 않습니다.

---

### 데몬 스레드

데몬 스레드는 사용자 스레드를 보조하기 위해 백그라운드에서 동작하는 스레드입니다.  

모든 사용자 스레드가 종료되면 데몬 스레드는 자동으로 종료되며, 주로 로그 처리, 자동 저장, 주기적 점검과 같은 부가 작업에 사용됩니다.  

데몬 스레드로 설정하려면 start() 호출 전에 setDaemon(true)를 호출해야 합니다.

---

### 스레드 생성 - Runnable 인터페이스 구현

Runnable 인터페이스를 구현하고 run() 메서드를 정의한 뒤, 이를 Thread 객체에 전달해 실행하는 방식입니다.  

이 방식은 클래스의 상속 구조에 구애받지 않고, 동일한 Runnable 객체를 여러 스레드에서 재사용할 수 있다는 장점이 있기에 Thread 클래스 상속보다 더 많이 쓰이는 방법입니다.
